# AI Moderation & Auto-Reply Guide

This project now supports:

1. Comment moderation pipeline:
- Sensitive-word check first (`comment-filter`)
- If hit words found, AI re-check decides allow/reject

2. Auto reply pipeline:
- Only comments explicitly marked `PASS` by `plugin-comment-moderation` are eligible
- Reply content is generated by AI (`auto-reply`)
- Built-in quota/rate controls to avoid burning daily credits
- If API quota/error happens, it silently skips reply (no "quota exhausted" text shown to visitors)

## 1) Required environment variables

Set these in server `.env` (or your secure secret injection path):

```env
MINIMAX_API_KEY=your_real_key
MINIMAX_API_URL=https://api.minimaxi.com/anthropic/v1/messages
MINIMAX_MODEL=MiniMax-M2.5
```

Optional controls:

```env
# comment-filter AI re-check
COMMENT_REVIEW_AI_ENABLED=true
COMMENT_REVIEW_AI_TIMEOUT_SECONDS=30
COMMENT_REVIEW_AI_MAX_CONTENT_CHARS=500

# auto-reply AI + quota controls
AUTO_REPLY_AI_ENABLED=true
AUTO_REPLY_AI_TIMEOUT_SECONDS=30
AUTO_REPLY_MAX_COMMENT_CHARS=180
AUTO_REPLY_MAX_REPLY_CHARS=120
AUTO_REPLY_DAILY_CALL_LIMIT=300
AUTO_REPLY_DAILY_AUTHOR_LIMIT=20
AUTO_REPLY_AUTHOR_COOLDOWN_SECONDS=60
```

If you are using pay-as-you-go MiniMax keys (non `sk-cp-`), you can still use:

```env
MINIMAX_API_URL=https://api.minimaxi.com/v1/text/chatcompletion_v2
MINIMAX_MODEL=MiniMax-Text-01
```

After changing `.env`:

```bash
docker-compose up -d --no-deps --build --force-recreate comment-filter auto-reply
```

## 2) Where to manage rules

### Auto-reply rules

Service: `auto-reply` (NOT Halo plugin settings)

APIs:
- `GET /api/v1/rules`
- `POST /api/v1/rules`
- `PUT /api/v1/rules/{id}`
- `DELETE /api/v1/rules/{id}`

Example:

```bash
curl -s http://localhost:8092/api/v1/rules
```

Create a rule (trigger + style hint):

```bash
curl -s -X POST http://localhost:8092/api/v1/rules \
  -H "Content-Type: application/json" \
  -d '{
    "name":"感谢类评论",
    "keywords":["谢谢","支持","喜欢"],
    "time_range":{"start":"00:00","end":"23:59"},
    "templates":["语气友好，简短致谢，并邀请继续交流"],
    "priority":100,
    "enabled":true
  }'
```

### Comment-filter rules (sensitive words)

Service: `comment-filter` (NOT Halo plugin settings)

APIs:
- `GET /api/v1/words`
- `POST /api/v1/words`
- `DELETE /api/v1/words`

Example:

```bash
curl -s -X POST http://localhost:8091/api/v1/words \
  -H "Content-Type: application/json" \
  -d '{"word":"敏感词示例"}'
```

Delete a word:

```bash
curl -s -X DELETE "http://localhost:8091/api/v1/words?word=敏感词示例"
```

## 3) Halo plugin settings

### Comment moderation plugin
- `enabled`
- `filterServiceUrl` (default `http://comment-filter:8091`)
- `filterStrategy` (`REJECT` recommended for strict moderation)

### Auto-reply plugin
- `enabled`
- `replyServiceUrl` (default `http://auto-reply:8092`)
- `replyAuthorName`

## 5) Quick test flow

1. Add one sensitive word in `comment-filter`.
2. Post a comment containing that word:
- harmless text should pass only when AI re-check returns allow.
- malicious text should stay blocked.
3. Add one auto-reply rule in `auto-reply`.
4. Post a normal comment that matches rule keywords:
- moderation plugin marks it `PASS`
- auto-reply plugin calls `auto-reply` service
- a delayed reply is created by ghost/system owner.
5. Hit limits (very long comment / repeated same author quickly):
- service should return `should_reply=false` silently.
