博客数据看板实现指南（Halo 2.x Public API 版）

适用于 Halo 2.x。该版本不再依赖旧的 Content API Access Key 开关，前端看板直接使用 Halo Public API。

一、项目目标

在 Halo 中创建公开访问的 /dashboard 页面，展示：

  • 博客统计（文章、评论、访问、分类、标签、运行天数）
  • 内容可视化（发布趋势、分类占比、热门文章、标签云）
  • 服务器监控（CPU、内存、磁盘、网络、实时曲线）

二、技术栈与约束

项目说明
博客引擎Halo 2.x
页面容器Halo Sheet 页面 + sheet_only_header_footer 模板
前端库jQuery（主题已内置）
图表库Chart.js 4.x（CDN）
布局CSS Grid + Flexbox + 响应式 + 深色模式适配
关键原则：前端页面不放任何 PAT/密钥。HALO_API_TOKEN 仅用于 GitHub Actions 在服务器端调用 Halo Console API。

三、数据源接口

3.1 Halo Public API（前端直接调用）

统一前缀：/apis

用途接口说明
站点统计GET /apis/api.halo.run/v1alpha1/stats/-返回站点访问/评论/文章/分类等总量
文章列表GET /apis/api.content.halo.run/v1alpha1/posts支持分页、排序；可提取趋势和热门文章
分类列表GET /apis/api.content.halo.run/v1alpha1/categories用于分类占比图
标签列表GET /apis/api.content.halo.run/v1alpha1/tags用于标签云与标签总数

3.2 Server Monitor API（自定义 Go 服务）

接口说明
GET /api/v1/metrics当前指标 + 5 分钟内历史
GET /api/v1/metrics/history最近历史点位
GET /api/v1/health健康检查

四、Nginx 配置（必须）

在博客域名的 server { ... } 中加入：

location /monitor-api/ {
    proxy_pass http://127.0.0.1:8093/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    add_header Access-Control-Allow-Origin * always;
    add_header Access-Control-Allow-Methods "GET, OPTIONS" always;
    add_header Access-Control-Allow-Headers "Content-Type, Authorization" always;

    if ($request_method = 'OPTIONS') {
        return 204;
    }

    limit_except GET OPTIONS { deny all; }
}

应用配置：

nginx -t
nginx -s reload

五、看板源码位置

完整可粘贴源码位于：

docs/dashboard-sheet-code.html
该源码已经改为 Halo 2.x Public API 调用方式，不需要填写 haloApiKey。

六、Halo 后台操作步骤

  1. 进入 Halo 后台 → 页面（SinglePage）→ 新建。
  2. 标题：数据看板，别名：dashboard。
  3. 模板选择：sheet_only_header_footer。
  4. 切换到源码模式，粘贴 docs/dashboard-sheet-code.html 全部内容。
  5. 发布页面，访问 /dashboard 验证。

七、GitHub Actions 与 PAT 说明

7.1 HALO_API_TOKEN 的用途

HALO_API_TOKEN 是 Halo Personal Access Token（PAT），仅用于工作流在服务器侧调用 Console API 进行插件安装/升级。

示例（服务器侧）：

curl -X GET "${HALO_URL}/apis/api.console.halo.run/v1alpha1/plugins" \
  -H "Authorization: Bearer ${HALO_API_TOKEN}"

7.2 存放位置建议

  • 推荐：GitHub Repository/Environment Secrets（当前做法正确）
  • 可选：服务器环境变量（仅服务器可读）
  • 禁止：写入前端页面源码、提交到仓库明文文件

八、安全建议

  1. 前端看板使用 Public API，不嵌入 PAT。
  2. 若你未来关闭 Public API 或改为私有接口，可通过 Nginx 服务端注入 Authorization 头，不要在 JS 中放令牌。
  3. /monitor-api/ 建议限制方法为 GET/OPTIONS，并保留访问日志。

九、故障排查

现象排查点
/dashboard 空白或图表不显示浏览器控制台报错、Chart.js CDN 是否可达、页面是否使用源码模式发布
监控卡片显示离线server-monitor 容器状态、/monitor-api/ 代理与 8093 连通性
博客统计加载失败确认 /apis/api.halo.run/v1alpha1/stats/- 与 /apis/api.content.halo.run/v1alpha1/posts 可访问
插件部署成功但页面没变确认 workflow 触发的是 Deploy Halo Plugins，并查看 Halo 插件升级日志
若你在 Halo 后台手工修改了同一页面内容，插件下次部署可能会覆盖（该页面属于插件托管页面）。
